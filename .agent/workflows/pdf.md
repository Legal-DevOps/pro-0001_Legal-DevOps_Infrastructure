Этот ворклоу активирует протокол "Nexus Prime DOC" для создания документов как глубокой компиляции всего контекста проекта.

### 1. Когнитивная подготовка (SKILL & KI First)

Прежде чем писать код — читаем инструкцию и доступные знания.

- `view /mnt/skills/public/docx/SKILL.md` — для архитектуры файлов.
- **Check KIs**: Проверка Knowledge Items на предмет ранее принятых архитектурных решений, чтобы избежать дублирования (DRY).

### 2. Стратегическое рассуждение (Strategic Reasoning)

Это этап "размышления повара перед готовкой". Мы не просто собираем факты, мы анализируем их через призму Байесовской логики:

- **Априорная цель**: Что пользователь просит буквально?
- **Скрытая цель**: Какую проблему он ПА САМОМ ДЕЛЕ хочет решить этим документом? (Напр. "убедить инвестора" или "зафиксировать баг").
- **Вероятностный стек**: Какой формат и тон (Tone of Voice) даст максимальный результат в текущих условиях?

### 3. Реконструкция контекста (Context Reconstruction)

Сбор "цифрового следа" из всей истории диалога:

- **Архитектурные ADR**: Какие решения (Rust, Python, Go) уже утверждены?
- **Эволюция идей**: Что вначале было хаосом, а теперь стало структурой?
- **Терминология**: Использование специфического сленга пользователя для создания ощущения "своего" продукта.

### 4. Состязательный самоанализ (Adversarial Self-Review)

Перед проектированием структуры мысленно атакуем свою гипотезу:

- "Этот документ слишком сложный?"
- "Не создаю ли я over-engineering там, где достаточно одной таблицы?"
- "Соответствует ли дизайн стандарту Luxury Tech или он выглядит дешево?"
*Если находим слабость — исправляем её еще до написания первой строчки кода.*

### 5. Проектирование скелета (Map Before Code)

Цепочка: **ЦЕЛЬ → СУЩНОСТИ → СТРУКТУРА → КОД → ФАЙЛ**.
Создаем PROJECT_MAP.md для документа, если он больше 10 страниц.

### 6. Программная сборка (Multi-Format Assembly)

Документ собирается как ПО. Текст и оформление — это данные, которые подаются на вход функциям генерации:

- **DOCX**: Для редактируемых отчетов с полной версткой.
- **PDF**: Для презентабельных, неизменяемых "Luxury" версий.

### 8. Финализация

- Проверка визуальной целостности (Premium Check) для всех трёх форматов (HTML, DOCX, PDF).
- Копирование в `PROJECT/outputs/`.
- Представление результата с кратким обоснованием принятых решений.
